# Korus Context

## Purpose
Runtime infrastructure module providing application context, bean management, 
dependency resolution, and lifecycle management.

## Characteristics
- **Runtime only**: Core runtime engine of Korus DI
- **Metadata consumer**: Uses metadata generated by core-processor
- **Container implementation**: Actual DI container logic

## Module Type
**Runtime Infrastructure Module**

## Dependencies
- `inject` - Core API definitions
- `slf4j-api` - Logging

## Usage
```xml
<dependency>
    <groupId>io.korus</groupId>
    <artifactId>context</artifactId>
</dependency>
```

## Content
This module will contain:
- ApplicationContext implementation
- Bean factory and registry
- Dependency resolution engine
- Scope management
- Lifecycle callbacks
- Proxy generation (if needed)
- Metadata loading and processing

## Design Principles
- **Build-tool agnostic**: No compile-time processing here
- **Metadata-driven**: Consumes metadata from core-processor
- **Runtime focused**: All runtime behavior
- **No processor dependencies**: Never depends on core-processor

## Scope Separation
Critical architectural boundary:
- **core-processor**: Generates metadata at build time
- **This module**: Consumes metadata at runtime

## Important
This module should NEVER have a compile dependency on `core-processor`.
The processor runs at build time and generates metadata that this module
loads and uses at runtime.
